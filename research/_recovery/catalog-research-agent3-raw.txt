## Overview

The MLB-YouTube dataset is a new, large-scale dataset consisting of 20 baseball games from the 2017 MLB post-season available on YouTube with over 42 hours of video footage.

## Dataset Components

The dataset consists of two components: segmented videos for activity recognition and continuous videos for activity classification.

**Segmented Videos:** The segmented video dataset consists of 4,290 video clips, with each clip annotated with various baseball activities such as swing, hit, ball, strike, foul, etc.

**Continuous Videos:** The continuous video dataset consists of 2,128 1-2 minute long clips from the videos, with each frame of the clip densely annotated with the baseball activities that occur, resulting in a total of over 15,000 activity instances.

## Dataset Challenges

The dataset is quite challenging as it is created from TV broadcast baseball games where multiple different activities share the camera angle, and the motion/appearance difference between the various activities is quite small.

The primary repository is available at **https://github.com/piergiaj/mlb-youtube** and contains source code, dataset and trained models.

## PitcherMotion Overview

PitcherMotion is a GitHub repository containing pose tracking data for baseball pitchers, with the main data file being Pitcher_Motion_Data.csv, which is a csv file of about 500 MB.

## Data Details

The dataset includes a companion csv file which contains Statcast data corresponding to the pitches included in this dataset. Data was taken for pitchers/pitch types which had been thrown at least 5 times at Tropicana Field in the 2021 regular season, with 5 sample videos used for each pitch (the last thrown by each pitcher), as Tropicana field was used because it has a centerfield camera angle which is easy to work with.

## Pose Tracking Methodology

All pose detection was done using KAPAO, an algorithm implemented in python. Detection of the pitcher was done automatically based on their position in the frame, and synchronisation of the pitches was performed automatically by using the pitcher's leg lift.

## Data Quality Flags

The dataset includes two reliability flags: "no_missing_frames" indicates continuous pitcher observation, and "smooth_CoM_flag" indicates stable torso position to avoid detecting different players as the pitcher.

## Pascale Walters' Baseball Tracking Project

Pascale Walters is a computer vision engineer and an MASc candidate in the Vision and Image Processing Lab at the University of Waterloo whose current research is around camera calibration and computer vision techniques for sports analytics.

### Project Overview

The baseball-tracking project uses a pre-trained YOLOv3 network (MS COCO dataset has a class for sports ball) because it is a small network that achieves good performance for detection.

### Key Technical Details

- **Ball Tracking**: The project assumes there is only one baseball in the video and defines the release point as the instant the bounding box of the baseball is above the player's bounding box.

- **Interpolation**: The ball is missed in some detections, which are interpolated quadratically before the release point and linearly after the release point.

- **Spin Analysis**: The spin axis is calculated by performing texture registration—a frame of the ball is warped to a frame of a ball in a later frame, using orthagonal projection and 3D rotation, and the best angle that represents this transformation is selected as the spin axis for the pitch.

### Output

The project generates a csv file of ball tracks with a column for the ball release point and produces a video with the bounding boxes of the ball, coloured blue if it is before the release point and yellow if it is after the release point.

## VideoPose3D Overview

VideoPose3D is a project by researchers including Dario Pavllo, Christoph Feichtenhofer, David Grangier, and Michael Auli that demonstrates 3D human pose estimation in video with temporal convolutions and semi-supervised training, published at CVPR 2019.

## Key Features

VideoPose3D is a lifting network where the input is 2D human pose and the output is 3D human pose, using dilated temporal convolution to exploit information across frames.

## WHAM Overview

WHAM is the official PyTorch implementation of "Reconstructing World-grounded Humans with Accurate 3D Motion." ## Key Features

- WHAM processes monocular video input to produce world-grounded 3D human motion, combining 2D pose estimation, camera motion tracking, and deep learning to recover 3D human pose and global trajectory using the SMPL body model.

- WHAM first detects 2D key points in the video frames using ViTPose, which will be useful for motion feature extraction. Because of object rotation and movement between frames, the key points are then normalized to a bounding box around the person, and motion context can be derived from them.

## Overview

AthletePose3D is a dataset and code for a CVSports at CVPR 2025 paper titled "AthletePose3D: A Benchmark Dataset for 3D Human Pose Estimation and Kinematic Validation in Athletic Movements".

## Dataset Details

The dataset includes 12 types of sports motions performed by athletes, comprising approximately 1.3 million frames and 165 thousand unique postures across disciplines such as running, track and field, and figure skating. Compared to conventional benchmarking datasets, AthletePose3D features notably higher speeds and accelerations, making it uniquely suited for athletic movement analysis.

## Key Findings

State-of-the-art monocular 2D and 3D pose estimation models trained on conventional datasets perform poorly on athletic motions; however, fine-tuning these models on AthletePose3D reduces the mean per joint position error (MPJPE) from 214mm to 65mm—a reduction of over 69%.

The work also validates the kinematic accuracy of monocular pose estimations through waveform analysis, highlighting strong correlations in joint angle estimations but limitations in velocity estimation.

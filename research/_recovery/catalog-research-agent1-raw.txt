# OpenPose Project Summary

**What It Does:**
OpenPose is a real-time computer vision library that detects human body positions and movements from images or video. It identifies keypoints across the entire body—including pose, hands, face, and feet—simultaneously for multiple people in a single frame.

**Key Details:**

- **Stars:** 33.9k
- **Primary Language:** C++ (88.7%)
- **License:** Free for non-commercial use; commercial licenses available
- **Last Update:** November 17, 2020 (v1.7.0)

**Core Features:**
- "Real-time multi-person system to jointly detect human body, hand, facial, and foot keypoints" totaling 135 keypoints
- 2D and 3D pose estimation
- Processing speed independent of the number of detected people for body/foot keypoints
- Cross-platform support (Ubuntu, Windows, macOS) with GPU and CPU options
- Python and C++ APIs for custom integration

**Technical Stack:** Built with C++, CUDA, and CMake; uses deep learning approaches based on peer-reviewed research published in IEEE TPAMI and CVPR conferences.

This makes it valuable for applications ranging from fitness tracking to motion capture and behavioral analysis.

# SAM 2: Segment Anything Model 2

## Repository Overview

**Description:** SAM 2 is a foundation model for promptable visual segmentation in both images and videos. It extends the original SAM with "a simple transformer architecture with streaming memory for real-time video processing."

**Star Count:** 18.9k stars

**Primary Language:** Python

**License:** Apache 2.0 (with exceptions for fonts)

**Last Update:** December 11, 2024 (most recent update regarding model compilation and multi-object tracking improvements)

## Key Features for Video Segmentation

SAM 2 delivers several powerful capabilities for video work:

1. **Real-time Processing** - The streaming memory design enables efficient frame-by-frame inference without requiring full video context at once.

2. **Multi-Object Tracking** - Users can prompt multiple objects independently and track them throughout video sequences. The latest update allows "relaxing the assumption of prompting for multi-object tracking and adding new objects after tracking starts."

3. **Interactive Refinement** - Supports adding prompts (points, boxes) on specific frames and propagating segmentations across the entire video automatically.

4. **Performance Optimization** - Recent updates include torch.compile support that delivers "a major speedup for VOS inference."

5. **Flexible Model Sizes** - Four checkpoint options (tiny through large) balance speed and accuracy for different applications.

The accompanying SA-V dataset represents the largest video segmentation dataset created to date, supporting the model's strong cross-domain performance.

# Pose2Par: Golf Swing Analysis Project

## Description
Pose2Par is an AI-powered system that converts standard video recordings of golf swings into comprehensive 3D biomechanical data. The project enables coaches and golfers to obtain professional-grade swing analysis without requiring expensive motion capture lab equipment.

## Key Metrics
- **Stars:** 9
- **License:** MIT (as indicated by badge)
- **Language:** Python 3.8+
- **Last Update:** December 16, 2024 (based on commit hash 6d02d76c)

## What It Does
The system automatically detects swing events, calculates joint angles, and measures golf-specific metrics including pelvis rotation, torso separation, and the X-factor (torso-pelvis angle differential). It processes standard video input through a containerized pipeline to generate frame-by-frame biomechanical analysis with interactive visualization.

## Techniques & Architecture
**Core Technologies:**
- **MMPose:** Deep learning-based 2D pose estimation for keypoint detection
- **LSTM/Transformer Models:** "AI marker enhancement" to correct tracking gaps and improve trajectory smoothness
- **OpenSim:** Physics-based biomechanical modeling for joint angle calculations

**Pipeline Flow:**
Video → Pose Estimation → ML Enhancement → Biomechanical Modeling → Analysis & Export

**Infrastructure:** Fully containerized with Docker Compose, supporting GPU acceleration via NVIDIA CUDA and multi-stage processing across separate microservices.

## Validation Status
The project includes independent validation results against lab-grade motion capture, showing high reliability for swing timing and pelvis rotation metrics, with documented limitations for certain measurements.
